Overview

Klaus (also known as Claws) is a sophisticated WhatsApp-based AI personal assistant built on the Warelay platform. It connects WhatsApp messaging to advanced AI agents, enabling intelligent, stateful conversations with tool access, memory persistence, and an extensible skill system.

WhatsApp Integration Layer

Klaus connects to WhatsApp through Baileys (WhatsApp Web).

Baileys WebSocket

  • Technology: Baileys library for WhatsApp Web scraping
  • Advantages: Fast setup, no external provider credentials
  • Use Case: Personal assistant scenarios with direct WhatsApp access
  • Session Management: Auth credentials stored in ~/.warelay/credentials/

Message Routing & Processing

The message router handles incoming WhatsApp messages through a structured pipeline:

flowchart LR A[Message Received] --> B[Parse Metadata sender body media group context] B --> C[Deduplicate by message ID] C --> D[Store Media to temporary disk] D --> E[Validation allowlist check] E --> F[Group Mention Check for klaus in groups] F --> G[Audio Transcription if configured] G --> H[Session Management load or create] H --> I[Agent Dispatch text command pi-agent] I --> J[Response Processing media optimization parsing] J --> K[Send Response with typing indicators] K --> L[Persistence save session state]

Memory Architecture

Klaus uses a layered, temporal knowledge graph-based memory system that replaces legacy recursive summarization.

flowchart TB subgraph MemoryLayers[Memory Layers] Working[Working Memory Current context window] Session[Session Memory Recent 25 conversation turns] Entity[Entity Memory Tracked entities with properties] LongTerm[Long-term Knowledge Decisions facts pending tasks] end Working --> Session Session --> Entity Entity --> LongTerm

Memory Storage Structure

~/klaus/whatsapp/{sessionName}/memory/
├── state.json                    # Session state + recent 25 turns
├── entities/
│   ├── index.json               # Fast entity lookup index
│   └── {entity_id}.json         # Individual entity files
├── knowledge/
│   ├── decisions.json           # Agreements, commitments
│   ├── facts.json               # Context, preferences
│   └── tasks.json               # Action items with status
└── relationships.json           # Entity-to-entity connections

Temporal Validity Model

All memory items include temporal metadata enabling time-travel queries:

interface TemporalValidity {
  validFrom: number      // When learned
  validUntil?: number    // When expired (null = still valid)
  source: string         // Origin (e.g., "extraction_...")
  confidence: number     // 0.0-1.0 score
}

Memory Query Tool Actions

  • search {query}: Find entities by name/alias
  • get {entityId}: Get full entity details with properties
  • history {entityId}: View property changes over time (temporal)
  • related {entityId}: Find connected entities
  • knowledge {type}: Query decisions, facts, or tasks

Tools & Extensions

Built-in Tools

Tools are created per-session via createTools(sessionName) to ensure proper sandbox isolation.

File Operations

  • read_file - Read files from scratchpad or system-wide
  • write_file - Write files to scratchpad
  • list_directory - List directory contents (system-wide access)
  • search - Grep-based pattern search in files

System & Web

  • bash - Execute shell commands (sandboxed via bwrap)
  • web_search - Search via Brave Search
  • web_fetch - Fetch and parse web content
  • get_current_time - Get current timestamp with timezone

AI & Media

  • generate_image - Text-to-image via Gemini
  • edit_image - Image editing via Gemini
  • mcp - Access MCP tool servers
  • whatsapp - Send WhatsApp reactions to messages

Background Tasks

  • start_grip - Start long-running grip (tmux or sleepless)
  • grip_status - List active grips
  • release_grip - Stop an active grip
  • grip_list_templates - List available grip templates

Scheduled Tasks

  • create_click - Create a new scheduled click (recurring task)
  • list_clicks - List all clicks for the current session
  • delete_click - Delete a click by ID

Memory

  • memory_query - Query entity-based memory system
  • memory_subscribe - Subscribe to memory updates
  • memory_unsubscribe - Remove a memory subscription
  • memory_share - Share entities across spaces

Session Management

Session Isolation

Each session gets its own tool instances with session context captured in closures:

// Each session creates its own tools with sessionCwd baked in
const tools = createTools(sessionName);  // e.g., "@17206598123"
// Tools use closure-captured sessionCwd, not global state

Session Key Derivation

function deriveSessionKey(scope: SessionScope, ctx: MsgContext): string {
  if (scope === "global") return "global";

  if (ctx.IsGroup && ctx.GroupJid) {
    if (ctx.GroupName) {
      return `#${sanitizeGroupName(ctx.GroupName)}`;  // #family-chat
    }
    return `#group_${extractGroupId(ctx.GroupJid)}`;  // #group_120363...
  }

  return `@${normalizeE164(ctx.From).replace(/^\+/, "")}`;  // @17206598123
}

Setup & Installation

Prerequisites

  • Node.js 22+
  • npm or pnpm package manager
  • Optional: API keys for model providers
  • Optional: Claude CLI for command mode
  • Optional: Tailscale for webhook setup

Installation Steps

# Global installation
npm install -g warelay

# Or from source
git clone 
cd warelay
pnpm install

# Link personal WhatsApp Web
warelay login

# Start relay with auto-reply
warelay relay --provider web --verbose

Configuration

  1. Copy .env.example to .env and configure API keys as needed
  2. Create warelay.json in ~/.warelay/ directory:
{
  "inbound": {
    "allowFrom": ["+12345550000"],
    "reply": {
      "mode": "command",
      "bodyPrefix": "You are a concise WhatsApp assistant.\n\n",
      "command": ["claude", "--dangerously-skip-permissions", "{{BodyStripped}}"],
      "claudeOutputFormat": "text",
      "session": { 
        "scope": "per-sender", 
        "resetTriggers": ["/new"], 
        "idleMinutes": 60 
      }
    }
  },
  "logging": {
    "level": "debug",
    "file": "/tmp/warelay/warelay.log"
  }
}
